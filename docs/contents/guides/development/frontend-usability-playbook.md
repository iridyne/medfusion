# 前端易用性落地手册（简单、实用、可执行）

> 文档状态：Beta
>
> 目标：指导团队用最小复杂度做出“用户能马上上手”的前端，而不是只做视觉层。

## 1. 先定义“易用”到底是什么

每个页面上线前，必须先写清 3 件事：

1. 用户来这个页面要完成什么任务
2. 完成任务的最短路径是什么（不超过 3 步）
3. 失败时用户下一步该做什么（不能只报错）

如果这三件事写不出来，页面就还不能叫“易用”。

---

## 2. 页面结构用同一个骨架

推荐固定成 4 层：

1. **意图层**：一句话说明这页做什么
2. **操作层**：主按钮 + 1 到 2 个次按钮
3. **反馈层**：状态、错误、可复制命令
4. **解释层**：边界说明和下一步

这套骨架比“堆控件”更稳定，用户也更容易形成操作记忆。

---

## 3. 交互规则（务必统一）

### 3.1 主按钮规则

- 每页只允许一个主动作按钮（`type="primary"`）
- 其他都用次按钮，避免“到处都是主操作”

### 3.2 错误反馈规则

- 只报“失败”不够，必须给修复建议
- 错误信息至少包含：
  - 发生了什么
  - 用户现在能做什么
  - 如果可定位，给“定位到对象”动作

### 3.3 状态反馈规则

- 所有异步操作必须有 `loading`
- 成功后给短反馈（如 “已连通”“已复制”）
- 长操作给阶段状态（检查中/已完成/失败）

---

## 4. 表单与配置页的最小约束

1. 默认值要可运行，不要空白开局
2. 参数分组按用户心智，不按后端字段顺序
3. 先校验阻塞项，再放行执行
4. 导出内容要可复制、可下载、可复现

推荐最小交付：`检查 -> 导出 -> 执行命令` 三连。

---

## 5. “简单实用”优先级清单

当时间不够时，按下面顺序做，不要反过来：

1. 跑通主任务链
2. 错误可修复
3. 状态可观测
4. 操作可回放（命令/配置可复制）
5. 最后才是视觉打磨

---

## 6. ComfyUI 集成页可复用模板

类似外部系统集成页（如 ComfyUI）建议固定成：

1. 地址输入 + 连通性检查
2. 一键打开外部系统
3. 最小上线链路（3 步）
4. 回到主链的明确 handoff 提示

这样可以把“能连上”与“能落地”区分开，避免只做跳转入口。

---

## 7. 上线验收清单

每个新页面在合并前至少自检：

- [ ] 新用户 3 分钟内能完成页面主任务
- [ ] 失败时能看到明确修复建议
- [ ] 至少有一个可复制命令或可下载配置
- [ ] 页面边界写清楚“做什么 / 不做什么”
- [ ] 对应 API 和前端状态字段已经对齐

满足以上 5 条，才算“可用”；否则只算“演示页”。
