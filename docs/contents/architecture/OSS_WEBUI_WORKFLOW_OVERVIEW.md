本页提供一张单页总览骨架图，用来快速解释 OSS WebUI 的整体结构、顶层 tabs 和各自承担的工作职责。

## 阅读方式

建议按以下顺序阅读：

1. 先看最上方的应用壳层，理解 `当前研究` 如何成为整个工作面的共享上下文。
2. 再看中间的 7 个 tab 小框，理解它们不是功能页集合，而是一条线性的研究主线。
3. 最后把这张图和 [MedFusion OSS WebUI 工作流导向改版 Spec](./OSS_WEBUI_WORKFLOW_SPEC.md) 对照阅读。

## 图中要点

- 顶层结构固定为：`概览 / 研究定义 / 数据 / 运行 / 验证 / 产物 / 系统`
- `概览` 负责开始和恢复上下文
- `研究定义` 负责生成研究合同
- `数据` 负责 readiness 检查
- `运行` 负责把合同解析成真实执行任务
- `验证` 负责给出研究结论
- `产物` 负责归档、导入、对比、导出
- `系统` 负责环境健康解释

## 使用边界

这张图主要用于：

- 设计讨论
- 信息架构对齐
- 前端改造前的范围确认
- 新成员理解 OSS WebUI 的产品方向

它不替代详细 spec，也不替代最终高保真视觉稿。
