# MedFusion 推广口播脚本

这份脚本是给两个平台共用的骨架。

原则：

1. 先讲痛点，不先讲技术名词
2. 先讲演示闭环，不先讲宏大平台愿景
3. 先讲真实结果页，不讲空壳页面
4. 明确当前是演示型 MVP，不夸大为完整生产平台

## 一、平台差异

### 小红书

- 建议时长：60 到 90 秒
- 表达重点：让人停留、看懂、觉得“这个东西拿来汇报很有用”
- 画面重点：模型库最新结果卡片、结果详情页、ROC / 混淆矩阵 / 注意力图
- 适合表达：导师汇报、组会展示、项目介绍、早期获客

### B 站

- 建议时长：3 到 5 分钟
- 表达重点：为什么做、现在做到哪、哪里是真能力、后面怎么补
- 画面重点：Slidev 结构说明 + Web UI Demo + 结果页 + 公开数据集入口
- 适合表达：工程化思路、validation、artifact、公开数据集验证路径

## 二、小红书 60 秒脚本

### 版本 A：结果页切入

开场：

`很多医学 AI 项目代码能跑，但一到给导师和医生演示的时候，就只剩命令行。`

中段：

`我最近把训练流程先收敛成了一个演示型 MVP。`

`这里可以先注册本地数据集，然后直接发起训练任务。`

`训练结束以后，不只是告诉你 done，而是会把结果页直接沉淀下来。`

`像 ROC、AUC、归一化混淆矩阵、注意力图、validation 摘要都会放到一个页面里。`

收尾：

`所以它现在更像一个适合导师汇报、组会展示和项目介绍的医学 AI Web 控制台。`

`如果你也在做这类项目，可以继续交流。`

### 版本 B：痛点切入

开场：

`你有没有发现，很多医学 AI 项目其实不是不能跑，而是没法讲。`

中段：

`我现在先不做大而全平台，而是把最关键的一条链先做通。`

`数据集管理，训练过程可视化，模型结果沉淀，最后能拿出真的结果图。`

`这套结果页里会有 ROC、混淆矩阵、注意力图，还有 validation 指标，不是纯 demo 图。`

收尾：

`这版先服务演示和传播，后面再继续补真实交付能力。`

## 三、B 站 3 到 5 分钟脚本

### 建议结构

1. 为什么做这个东西
2. 为什么命令行 demo 不够
3. 当前 MVP 的边界
4. 现场走一遍闭环
5. 重点展示结果页
6. 说明 validation 和 artifact 不是假的
7. 讲公开数据集怎么快速验证
8. 讲下一阶段 roadmap

### 参考讲法

第一段：背景

`我最近在做一个医学 AI 的 Web 化演示控制台。`

`原因很简单，很多项目训练代码其实能跑，但一到导师汇报、医生沟通或者对外介绍的时候，就很难讲清楚。`

第二段：当前边界

`所以这一版我没有继续堆功能，而是把目标限定成演示型 MVP。`

`只保留数据集、训练监控、模型库和结果页这条主链。`

第三段：演示闭环

`第一步注册数据集。`

`第二步直接发起训练。`

`第三步看训练过程。`

`第四步看训练结束后沉淀下来的模型记录和结果页。`

第四段：结果页重点

`这里我现在重点补的是结果部分。`

`除了训练曲线，还会有 ROC、归一化混淆矩阵、阈值分析、校准摘要、误分类摘要，还有多模态注意力图。`

`这样对外演示时，别人看到的不是“训练跑完了”，而是“这个结果到底长什么样”。`

第五段：可信度

`我现在也在把 validation 和 artifact 这条线补实。`

`也就是说，前端展示的这些图和指标，后面都有对应的 artifact 文件和 validation 摘要，不是写死在前端的 demo 内容。`

第六段：公开数据集入口

`如果你现在还没有私有数据，也可以先用公开数据集验证。`

`比如 MedMNIST 适合最快跑通，UCI Heart Disease 适合表格任务，ISIC 或 ChestXray14 更适合做公开医学影像验证。`

第七段：roadmap

`后面我还想做一个更吸引人的方向，就是参考 ComfyUI 的节点式拖拽搭建。`

`但这个方向我会放在主链稳定之后，不会先做一个看起来很炫但后端跑不通的壳。`

## 四、录屏结构建议

最稳的方式不是全程录浏览器，而是：

1. 先用 Slidev 讲 3 到 5 张关键信息页
2. 再切到 Web UI 演示闭环
3. 再回到 Slidev 总结

建议顺序：

1. 封面
2. 痛点
3. 这版 MVP 的边界
4. 闭环四步
5. 切到 Web UI
6. 展示结果页
7. 回 Slidev 讲公开数据集入口
8. 讲 roadmap 和 CTA

## 五、标题草案

### 小红书

1. `还在用命令行给导师演示医学 AI 训练？`
2. `我把医学 AI 训练流程做成了一个可演示的 Web 控制台`
3. `医学 AI 项目最难的不是跑，是怎么讲清楚`
4. `这个结果页，终于能把医学 AI 项目讲明白了`

### B 站

1. `我为什么做一个医学 AI 的 Web 演示控制台`
2. `把医学 AI 训练流程做成 Web UI 之后，演示终于顺了`
3. `MedFusion 演示型 MVP：从数据集到结果页的闭环`
4. `医学 AI 项目，不该只剩命令行可以展示`

## 六、录制前检查清单

- 前端首页默认落在可演示页面
- 模型库里已有可展示的最新模型
- 结果详情页能稳定打开
- ROC、混淆矩阵、注意力图都能正常显示
- 公开数据集入口已经写进 README
- 口播时不说“完整平台”“生产可用”“全自动”

## 七、当前最建议先录的两条内容

1. `结果页展示型`
   重点展示结果页和 artifact，最适合小红书冷启动

2. `公开数据集验证型`
   重点展示“没有私有数据也能自己试”，最适合做 GitHub README 和 B 站转化
